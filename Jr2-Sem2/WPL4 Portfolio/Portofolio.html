Inleidende tekst

Als toekomstig programmmeur hou ik van uitdagingen en denk ik graag probleemoplossend na. Naast een kritische blik ben ik ook sterk in het analyseren van mogelijke problemen en kan ik mezelf sturen om tot oplossingen te komen. Daarnaast leer ik ook graag bij wat bijdraagt aan mijn competentie om alles wat mij voorgelegd wordt tot een goed einde te brengen.

Waarom studeer ik programmeren?

Ik heb gekozen om een opleiding in programmeren te beginnen om verschillende redenen. Een meer praktische reden is om een diploma op de hand te hebben. Ik heb tijdens de 5 jaar sinds ik gestopt ben met studeren lang nagedacht of ik nog zou studeren en zoja wat ik dan zou studeren. Programmeren was een van het enige waar ik mij nog aan zou wagen.

Tijdens een van mijn vorige studies was ik ook al in aanraking gekomen met programmeren, toen had ik al gevoel dat ik het wel fijn vond en het ook redelijk goed kon. Ik hou van het meer analytisch nadenken en oplossen van problemen en omdat code altijd beter kan is hier nooit een tekort aan. Een diploma in programmeren geeft natuurlijk ook vele toekomst opties aangezien bedrijven ze meer en meer nodig hebben.

Waarom past een job in programmeren bij mij?

Een job in programmeren past mij voor veel van de zelfde reden dat ik deze opleiding gekozen heb. Zo vind ik programmeren en de uitdagingen die hier bij tepas komen fijn. Dit zal natuurlijk gedeeltelijk afhangen van waar ik een job vind. Mijn sterkes liggen ook veel meer in analytische en back-end oplossingen tegenover people management. Hoewel people management in elke job een belangrijke factor is, is het in een programmeer job toch van minder belang.

Comptenties Programmeur

Ontwerper

Ik kan al redelijk goed inschattingen maken over hoe haalbaar een project is. Een voorstel maken op basis van standaarden en methodiek is moeilijker. Zo heb ik nog geen goed beeld van verschillende methodieken en hun voor- en nadelen.

    Voorbeelden:
  1. Bij de eerste oefeningen bij C# was het doel enkel enkele functionaliteiten uit te werken
  2. Bij Galgje was het belangrijk om direct een goed idee te hebben waar dit programma naartoe kon gaan en zo te zorgen dat er uitbreiddingen bij konden komen zonder opnieuw te moeten beginnen
  3. Bij latere oefeningen in C# waren uitbreiddingen minder belangrijk maar bleven voordien geleerde functionaliteiten nog altijd belangrijk. Zo was layout bij Eeve catcher nog altijd belangirjk
  4. Toen ik hulp vroeg aan vrienden bij vroege oefeningen stelden ze vaak nieuwe klasses of arrays/lists voor terwijl we deze nog niet hadden gezien. Ik heb dit toen niet gevolgd omdat dat niet bij de methodieken die ik toen moest gebruiken hoorden.
  5. Bij Advent of Code problemen die ik af en toe heb gemaakt tijdens mijn vrije tijd was het enige belangrijke een snel genoeg runtime en de juiste oplossing. Uitbreidding of leesbaarheid was niet belangrijk

Programmeur

Over het algemeen denk ik dat ik goed en efficient code kan schrijven. Mijn algemene structuur van code zit ook redelijk goed alhoewel dit nog zeker beter kan. Ik probeer altijd snelle en efficiënte code te schrijven maar hier weet ik vaak niet wat snel en goed is en wat niet. Ik heb nog geen goed beeld van hoe de C# compiler werkt en hoe ik hier beter mee kan werken Ik ben ook niet zeker over 'naming conventions', alhoewel ik de standaard camel casing en pascal casing goed toe pas weet ik niet of mijn functie en variabelen namen leesbaar zijn. Ook weet ik niet of mijn persoonlijke methode om xaml objecten te benoemen een goede methode is.

    Voorbeelden:
  1. De (bijna) volledige code van mijn galgje project.
  2. Verschillende Advent of Code oplossingen.
  3. Gedeeltelijk worms spel in java (enkel code zelf, geen lay-out).
  4. 2D star trek spel puur in console.
  5. Vele oefeningen bij C#, bijvoorbeeld subsets van alle mogelijk sets uitprinten.

Tester

Hoewel ik tijdens het eerste sememster al veel mijn eigen en andere mensen hun code getest heb, geloof ik dat ik er toch niet zo ver in sta. Alle testen die ik gedaan heb waren simpelweg het programma manueel uitproberen en manueel 'edge cases' vinden. Ik heb dit jaar nog nooit zelf tests geschreven.

    Voorbeelden:
  1. Zelf testen van oefeningen die ik gemaakt heb en kijken of ze de juiste uitkomst hadden, zelfs bij extreme input.
  2. Bij Advent Of Code moest ik veel data gebruiken dus had ik zelf een kleinere dataset gemaakt om mijn code te testen en te checken.
  3. Voor Galgje heb ik veel getest, door extreme waarden in te geven (lange woorden met gekke tekens) en op onlogische momenten alle knoppen proberen in te duwen.
  4. Ik heb ook Galgje van een paar medestudenten getest.
  5. In een vorige opleiding heb ik voor veel oefeningen en een eindproject worms wel zelf testen geschreven en uitgevoerd.

Communicator/Teamspeler

Tijdens deze opleiding heb ik nog niet direct samengewerkt met iemand anders. Wel heb ik al regelmatig een paar medestudenten geholpen met C# (en soms hun met mij). Wel heb ik tijdens een vorige opleiding een groot project met 2 samen afgewerkt. Ik denk dat ik het vrij goed stel als teamspeler ondanks het feit dat ik hier het laatste jaar nog weinig ervaring mee heb gehad.

    Voorbeelden:
  1. Het helpen van medestudent bij oefeningen van C# en Galgje. Ik heb hier hun bij geholpen en vaak mezelf ook een beetje door het beter te begrijpen.
  2. Het maken van 2 Advent Of Code opdrachten met een vriend.
  3. Worms als eindopdracht in een vorige opleiding. Deze opdracht ging op zichzelf voor de volledige 6 studiepunten van dat vak en moest samengemaakt worden met 1 medestudent. Hier heb ik bijna 100 uur samengewerkt met iemand.
  4. Hulp die ik heb gekregen bij het maken van mijn galgje project.
  5. Voor sommige vroege oefeningen van SQL heb ik samengewerkt met een medestudent omdat we ze zelf nie alleen konden maken.

Levenslang lerende IT-Professional

Ik geloof dat levenslang leren een van mijn sterkste punten is. Zelfs in het eerste semester van deze opleiding heb ik al zelfstandig, of met hulp van vrienden, veel nieuwe dingen geleerd die ik tijdens deze opleiding kan gebruiken. Ik ben ook redelijk leergierig en zal dus zelf vaak nieuwe informatie en uitdagingen opzoeken.

    Voorbeelden:
  1. Het zelf maken van een object voor het galgje project heb ik zelf uitgezocht nadat een vriend had gezegd dat hij het op deze manier zou oplossen.
  2. Voor Advent Of Code heb ik zelf nieuwe dingen moeten opzoeken om de opdrachten zelfs te kunnen oplossen
  3. Ik heb animaties gebruikt voor mijn galgje project, ook al hebben we dit nog niet gezien tijdens de les
  4. Ik kijk of lees af en toe over coderen. Een voorbeeld hiervan is hoe branchless programming werkt en de voordelen en valkuilen van deze techniek
  5. Ik ben op mijn 28ste terug naar school gegaan, zelfs na een korte werk carriere.

Analyse arbeidsmarkt

Voor de opdracht 'Analyse arbeidsmarkt' moesten we 5 verschillende vacatures vinden waar wij zouden kunnen soliciteren na deze opleiding. Bij elke vacature moesten we een kleine beschrijvingen geven van verschillende elementen in deze vacature. Zo moesten we naast wat basisinfo zoals de naam/locatie van het bedrijf en plaats van vacature ook het type bedrijf en de bedrijfscultuur te weten komen. Hiernaast moesten we ook persoonlijke info bij deze vacature geven zoals de sterke kant en zwakke kant van de Job

De grootste persoonlijke uitdaging hier was kiezen tussen al de mogelijke vacatures. Wat ik heb bijgeleerd is dat er heel veel vacatures voor programmeur te vinden zijn. Daarlangs heb ik ook een paar termen voor het eerst gezien die later ook in andere lessen terug kwamen zoals SCRUM.

SeminarieVerslag

Voor het seminarieverslag moesten we een paar kleine vragen beantwoorden die te maken zouden moeten hebben met 2 lezingen de we gekregen hebben tijdens WPL. Zo ging een lezing over git en moesten we daarna de werking en voordelen van git uitleggen. De andere lezing ging over de verschillende soorten jobs als programmeeur.

Deze opdracht vond ik persoonlijk het moeilijkste. Ik had problemen met het linken van de vragen met de lezingen. Zo was er een vraag over welk bedrijf mij het meeste aansprak hoewel de lezingen nauwelijks of niet over specifieke bedrijven gingen. Het enige wat ik dan ook mogelijks heb bijgeleerd is veel kunnen vertellen over heel weinig. Hoewel dit op het eerste zicht onzin lijkt ben ik er wel van overtuigd dat dit belangrijk kan zijn om carrière te maken in een later bedrijfsleven.

Galgje

Voor de opdracht Galgje moesten we met C# en wpf een galgje applicatie maken. De project requirements voor deze applicatie werden in 3 delen aan ons gegeven. Deze moesten analoog zijn alsof een klant ze aan ons gaf. Het was belangrijk dat we aan al deze project requirement voldeden zonder al te veel extra bij te coderen. Uiteraard moest de applicatie ook werken en zonder bugs zijn.

Echt grote specifieke uitdagingen had ik niet bij dit project. Al de uitdagingen die ik zag waren heel vergelijkbaar met uitdagingen die ik had bij andere C# oefeningen. De meeste van deze kleine uitdagingen werden uiteindelijk ook iets dat ik leerden. Zo had ik weeral problemen met images te laden maar heb ik nu wel tegoei door waar dit aan lag. Ik heb ook terug na 10 jaar weer eens zelf een object aangemaakt en gebruikt omdat dit mij veruit de simpelste manier leek om een bepaald probleem op te lossen. Zo zjin er nog een paar andere voorbeelden.

CV

Voor de opdracht CV moesten we via een website onze CV maken. We mochten zelf kiezen hoe deze CV eruitzag, zolang hij maar professioneel genoeg was.

Ook hier had ik geen echte grote uitdagingen, en het proces van deze uitdagingen waren heel vergelijkbaar met die bij galgje. Zo eindigde de meeste uitdagingen met iets dat ik geleerd heb. Zo heb ik transforms gebruikt in css om mijn image uit mijn pagina te laten schieten bij het hoveren ervan. Ook heb ik bij deze opdracht voor het eerst pas tegoei een grid gebruikt om een juiste volgorde van mijn divs te verkrijgen.

Individuele opdracht 1

Voor de eerste individuele opdracht die we tijdens WPL2 moesten maken, moesten we 2 klasses in C# aanmaken. Een eerste klasse was VoornaamNaam. Deze klasse moest een paar properties hebben die over jouw gingen, inclusief Hobbies. Hobbies was de 2de klasse de we moesten maken. Deze klasse werd een object van een property uit de eerste klasse.

Puur inhoudelijk had ik met deze opdracht heel weinig problemen. Wel was dit de eerste keer dat we met meerdere mensen tegelijk in 1 git repository moesten werken. Een bijkomend probleem was ook direct dat vele klasses dezelfde naam hadden en zo problemen konden veroorzaken met namespace. Het meeste heb ik dus bijgeleerd over git en namespace conflicten.

Individuele opdracht 2

De tweede individuele opdracht was een kleine web opdracht. We moesten een productcard maken van een product naar keuze. We mochten deze maken en stylen zoals wij zelf wouden. Ik heb een productcard gemaakt van een gitaar die ik op dat moment mogelijks wou kopen.

Deze opdracht was meer een opfrisser van het vorige semester. Het had enkel standaard css en html nodig die we al in semester 1 gezien hadden.

Individuele opdracht 3

Individuele opdracht 3 was weer een C# opdracht. Voor deze opdrachten konden we kiezen tussen 1 van 5 klasses om te maken. De 5 klasses waren Designer, Scrummaster, Webdeveloper, C#developer en Tester. Ik heb de klasse tester gemaakt. Deze klasse had een paar simpele methodes nodig zoals bijvoorbeeld 'Divide' wat simpelweg 2 getallen door elkaar moest delen. Wat extra bijkwam bij deze klasse was dat het ook gebruik moest maken van exceptions en 'throw-catches'.

Exception en throws was iets dat ik op het moment van deze opdracht zeker nog niet onder de knie had. Tijdens het maken van deze opdracht heb ik dan ook een paar belangrijke stappen gezet in het beter begrijpen van hoe en wanneer ik try, throws, exceptions en catches het best gebruik.

Hoofdproject

Voor het hoofdproject van wpl2 moesten we een verhuursite voor een product naar keuze maken. Wij hebben gekozen om elektrische voertuigen te verhuren. De Frontend en de Backend zijn hieronder te downloaden. We hebben alle backlogitems (te vinden in het logboek) kunnen vervullen.

De site heeft een landingpage, een profielpagina waar je een account kunt aanmaken en kan inloggen, een productpagina waar alle producten op te vinden zijn je deze kan sorteren/filteren. Vervolgens heeft onze website ook een individuele productpagina met meer info over 1 specifiek product en een knop om deze te huren voor bepaalde datums. Ten laatste heeft de website ook nog een functionele winkelmand en bestellingshistorie. Bijna alle data wordt ingeladen via de database, enkel de winkelmand maakt gebruik van de localstorage.

Tijdens het eerste semester van deze opleiding heb ik opnieuw gemerkt dat mijn sterktes vooral bij hard skills liggen. Zo heb ik weinig tot geen problemen met puur C# en SQL coderen en weinig problemen met html en css. Ook analytisch en probleem oplossend denken blijfen sterke punten van mezelf.

Ik ben minder sterk in de meeste soft skills maar toch zijn er een paar waar ik relatief goed in ben. Zo denk ik dat ik een vrij sterke teamspeler ben en zeker goed ben in andere mensen helpen met hun code. Ook denk ik dat ik heel zelfstandig kan werken en leren.

In andere soft skills ben ik dan weer veel slechter, zoals communicatie met klanten of, zoals ik hier moet doen, mezelf verkopen of uitten via een CV of portfolio. Mezelf naar de buitenwereld verkopen en tonen is al lang een werkpunt van mezelf, en dat zal het waarschijnlijk nog lang zo blijven.

Wat vond je van het project?

Hoewel ik heel veel heb geleerd uit dit project heb ik toch het gevoel dat het veel te ambitieus was (en vooral veel te vroeg in onze ontwikkeling). Zo hebben bijvoorbeeld heel veel communicatie tussen de front-end en backend gedaan wat we zelfs pas volgend jaar gaan zien. Dit zorgde het meest voor frustratie in bugs.

Heb je veel steun gehad van je team/lectoren?

Ik probeer persoonlijk zo weinig mogelijk aan lectoren te vragen en het vooral zelf uit te zoeken, of het aan mijn collega’s te vragen. Het internet, mijn team en kameraden hebben mij veel vooruitgeholpen. In de uitzonderlijke situatie dat ik een vraag stelde aan een lector, werd het ook goed beantwoord.

Wat heb je bijgeleerd op technisch vlak?

Zowat alles dat ik momenteel ken van JavaScript, Sql Integratie in .Net en hoe controllers werken en websites in het algemeen.

Welke soft skills heb je zien groeien bij jezelf?

Op grote schaal denk ik niet dat mijn soft skills verbeterd zijn. Natuurlijk heb ik doorheen het project wel beter met mijn team leren communiceren en werken, maar dit zal waarschijnlijk zo bij elk nieuw team opnieuw moeten gebeuren. Elk team is anders.

Wat zou je anders aanpakken in je volgend team?

Ik kende de mensen in dit team al heel goed voor we eraan begonnen (behalve Joep). In een nieuw team dat ik minder goed ken, ga ik iedereen meer behandelen zoals ik Joep nu heb behandeld.

Als je morgen opnieuw zou mogen beginnen, wat zou je anders doen?

Meer structuur in de JavaScript brengen, door ofwel een framework te gebruiken, ofwel één JavaScript per pagina en een andere JavaScript als library te gebruiken. Doordat we met weinig voorkennis met meerdere personen aan meerdere JavaScripts zijn begonnen, was het moeilijk om een duidelijke structuur te behouden.

Eindreflectie

Positieve realistaties

Ik was snel door de kleinere projecten heen die ze op voorhand voor mij voorbereid hadden. Al snel werden de projecten minder duidelijk van omvang. Zo moest ik in de 3rde week al kijken of ik sharepoint sites kon automatiseren met Microsoft Graph. Dit terwijl MS Graph nauwelijks in publieke beta zat.

Positief op de werkplek

Het positiefste op de werklep vond ik de collega's. Het zijn stuk voor stuk fijne mensen

Moeilijke punten

Het moeilijkste was communicatie met mensen die niet op de It afdeling zelf werkte. Ik kon ze enkel contacteren via mail. Ik moest vragen software gerelateerde vragen stellen aan mensen waarvan ik geen idee had hoe goed ze deze zouden begrijpen.

Samenwerking

Het meeste van mijn tijd heb ik alleen moeten werken. 3 van de 4 projecten waren stukken software waar ik zelf aan werkte. Bij 1 van deze 3 was een beetje contact nodig met iemand van de HR afdeling voor wie ik deze software aan het schrijven was. Voor het 4de project moest ik wel samenwerken met iemand om een deel van scanapp die hij aan maken was bij te helpen. De samenwerking hierbij ging heel vlot, zo heb ik algemene services aan zijn software kunnen en mogen toevoegen.

Beeld van de job

Over het algemeen kwam het heel erg overeen met mijn beeld. Ik heb al een paar jaar werkervaring bij welke ik al in contact kwam met Profel zelf. Ook heb ik een paar vrienden die in de software wereld werken.

Vooruitblikken op WPL4

Ik hoop meer en beter te kunne communiceren met andere mensen binnen profel, vooral mensen buiten de It dienst zelf. Betere communicatie zal zorgen dat ik de juiste data sneller en beter krijg en ook een beter beeld heb van wat er van de software verwacht wordt. Toch hoop ik iets meer uitgelijnde taken te krijgen waar ik zelf niet te veel achter ga moeten zoeken wat ik precies moet doen. Het meeste tijd bij Profel heb ik besteed aan uitzoeken hoe bepaalde technologieën werken en hoe ik deze kan gebruiken in plaats van functionele software te schrijven.
